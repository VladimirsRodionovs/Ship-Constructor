# Контекст разработки

Проект: Java + Maven, разработка в IntelliJ IDEA.

## Правила работы

1. Никаких правок и написания кода без разрешения пользователя.
2. Сначала согласовываем изменения, потом выполняем.
3. Любые изменения в проекте делаются только после явного подтверждения пользователя.

## Примечание

Этот файл фиксирует рабочие правила взаимодействия для текущего проекта.

## Текущий контекст (срез на 2026-02-28)

- Основной фокус: конструктор модулей + конструктор каркаса корабля.
- БД: MySQL `EXOLOG`, таблицы `ShipModules` и `SpaceShips` (подключение через внешний env-файл).
- Инструмент сейчас: внеигровой редактор (JavaFX), локальное сохранение JSON + частичная работа с БД.
- Подход по каркасу: `идеал -> деградация` с инженерными коэффициентами и ограничениями.

## Зафиксированная выполненная работа

1. Module Designer (JavaFX)
- Вкладки формы + локальное сохранение/загрузка JSON.
- `Save to DB`/`New` (поиск следующего свободного `ModuleID`).
- Упрощенная валидация (по согласованию) + предупреждения для шасси.
- Добавлены `SizeType/SizeTotal/SizeDimentions`, `StructureProfileCode`.
- Добавлены A-F параметры шасси и выбор режимов оптимизации/полосы запаса.

2. Каркас расчета шасси
- Введены load-cases, capability-классы, thrust/mass модели.
- Добавлены коэффициенты материалов/структур/технологий (`A-F`) и trace-диагностика.
- Введены `axial/bending/torsion` диагностические доли нагрузки.
- Добавлены режимы оптимизации: `MIN_MASS`, `BALANCED`, `HIGH_MARGIN`.
- Добавлена целевая полоса запаса (`target min/max margin`) и диагностические предупреждения.

3. StructuralGraph и визуализация
- Построение структурного графа: nodes/members/panels.
- Планирование нагрузок и pre-solver envelopes (members + panels).
- Профиль-зависимое распределение `members/panels` и ролей элементов.
- Убраны сквозные диагонали по умолчанию, добавлена метрика вторжения во внутренний объем.
- В `Chassis Graph` есть 2D и 3D wireframe (вращение/zoom), плюс lightweight pass.

4. Размещение двигателей (новый этап)
- Введен составной `EngineLayoutConfig` (слои, паттерны, поворот плоскости, симметрия).
- Приоритет генерации: `ThrustLayout -> EngineLayoutConfig -> legacy profile fallback`.
- В UI добавлен `Engine Layout Builder` (добавление/удаление слоев, синхронизация с JSON).

## Что дальше (ближайший шаг)
